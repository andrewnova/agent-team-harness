How It Works

One harness. Two teammates. One source of truth.

Agent Team Harness keeps goals, tasks, leases, reviews, mailbox messages, browser proof, computer proof, and closeout records in local project state. Models can collaborate without making chat history the system of record.

01

Plan

Codex and Claude both weigh in before tasks are created.

02

Route

Frontend work goes to Claude; backend and proof stay with Codex.

03

Review

Each model checks the other before merge gates open.

04

Verify

Browser, computer-use, and command proof attach to task state.

Built For Real Work

Designed to avoid the expensive failure mode: two smart models chatting forever.

Nonblocking Claude work

Codex keeps working while Claude handles frontend implementation, critique, or long-context review.

Semantic acknowledgements

Requests require meaningful replies: what was received, what happens next, and what is blocked.

Closeout reports

Completion can include tasks, approvals, proof artifacts, warnings, screenshots, and self-heal notes.

Install

Clone, install, and start from any project.

git clone https://github.com/andrewnova/agent-team-harness.git
cd agent-team-harness
./scripts/install-codex.sh

cd /path/to/your/project
agent-team doctor --fix --target my-project
agent-team start --name my-project --project-dir "$PWD" --daemon
